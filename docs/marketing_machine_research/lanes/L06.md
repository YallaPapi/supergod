# L06 - Measurement System

## 1) Three options in-lane

### Option A: Lean Last-Touch + Self-Reported Attribution (fastest)
**Attribution model**
- Primary: `self-reported attribution` field on intake form ("Where did you hear about us?") plus manual normalization.
- Secondary: HubSpot `Original Traffic Source` and `Latest Traffic Source` for platform-captured channel context.
- Decision rule: if self-reported source conflicts with cookie source, prioritize self-reported for channel planning (dark social/referrals are often missed by clickstream) `UNPROVEN`.

**KPI definitions**
- `New leads/week`: count of qualified inbound + outbound replies that meet ICP.
- `Booked call rate`: booked calls / qualified leads.
- `Show rate`: attended calls / booked calls.
- `Close rate`: paying clients / attended calls.
- `CAC (cash)`: (ad spend + contractor prospecting cost) / new paying clients.

**Experiment loop**
- 2 experiments/week, one variable per test (offer angle, CTA, creative hook, outreach opener).
- Use sequential comparison (current control week vs next variant week) instead of strict significance at very low volume.

### Option B: Native Multi-Touch Reporting (GA4 + HubSpot Attribution)
**Attribution model**
- GA4 attribution reports using available models (data-driven, paid+organic last click, Google paid channels last click).
- HubSpot attribution reports for contact/deal creation and revenue contribution.
- Decision rule: budget shifts require agreement between at least 2 lenses (e.g., GA4 assisted lift + HubSpot deal attribution).

**KPI definitions**
- `Assisted conversions` by channel.
- `Cost per SQL` by channel.
- `Pipeline $ influenced` by campaign.
- `Revenue per content asset` (attribution report driven).

**Experiment loop**
- Weekly readout from GA4 + HubSpot attribution dashboard.
- Keep only channels/assets that produce attributable SQLs and deals over 2-week windows.

### Option C: Experiment-First Incrementality (holdout/control heavy)
**Attribution model**
- Minimal deterministic attribution; rely on controlled holdouts (audience split/withheld segment) to estimate incremental impact.
- Treat click attribution as directional only.

**KPI definitions**
- `Incremental booked calls per 100 prospects reached`.
- `Incremental cost per booked call`.
- `Lift %` versus control over fixed test window.

**Experiment loop**
- Always-on A/B or control-vs-variant structure with predefined stop rule.
- Require clear practical effect size before rollout; otherwise keep control.

## 2) Option comparison snapshot
- Option A: fastest setup, lowest tooling friction, moderate attribution accuracy.
- Option B: richer attribution views, medium setup complexity, stronger reporting depth.
- Option C: strongest causal rigor, slowest to operate for small volumes.

## 4) Key failure modes + mitigations
- **Failure mode:** channel misattribution from dark social, DMs, word-of-mouth.
  - **Mitigation:** mandatory self-reported source field + weekly normalization dictionary (e.g., "reddit", "Reddit post", "subreddit").
- **Failure mode:** noisy experiment conclusions from tiny sample sizes.
  - **Mitigation:** use minimum decision windows (7 days) and effect-size thresholds (e.g., >=20% lift before rollout) `UNPROVEN`.
- **Failure mode:** KPI sprawl (too many metrics, no decisions).
  - **Mitigation:** freeze to 5 core KPIs for first 14 days; every KPI must map to a weekly keep/kill/change decision.
- **Failure mode:** CRM hygiene drift (missing lifecycle stages, incomplete cost fields).
  - **Mitigation:** daily 10-minute data QA checklist and required-property validation before stage transitions.

## 5) Concrete 14-day execution outline
- **Day 1:** define canonical funnel stages (Lead -> Qualified -> Booked -> Attended -> Closed Won/Lost) and KPI dictionary in one shared doc.
- **Day 2:** add intake field for self-reported source; add controlled source-value list; enforce required capture.
- **Day 3:** configure HubSpot source properties views, saved filters, and one master pipeline board for stage counts.
- **Day 4:** add cost capture fields (ad spend, tooling, contractor time proxy) and CAC calculation field.
- **Day 5:** build a single weekly scorecard (5 KPIs + top 5 source buckets + top 5 offers/messages).
- **Day 6:** baseline pull: current conversion rates by stage and current top channels.
- **Day 7:** launch experiment 1 (offer positioning A vs B) with one variable changed.
- **Day 8:** launch experiment 2 (outreach opener A vs B) for outbound sequences.
- **Day 9:** run data QA; fix uncategorized sources; reclassify ambiguous entries.
- **Day 10:** mid-test review: keep tests running unless severe underperformance (>30% down) `UNPROVEN`.
- **Day 11:** close week-1 tests; decide keep/kill/change per experiment.
- **Day 12:** ship winning variant(s) to 100% of relevant traffic/outreach.
- **Day 13:** start next 2 tests from biggest bottleneck stage (usually Lead->Booked or Booked->Attended).
- **Day 14:** full retro: update KPI baselines, document learnings, lock next 14-day test backlog ranked by expected booked-call lift.
