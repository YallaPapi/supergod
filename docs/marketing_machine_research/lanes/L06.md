# L06 — Measurement System

## 1) Three in-lane options

### Option A — Lean deterministic attribution (recommended for first 10 clients)
**What it is**
- Use strict UTM tagging on every inbound link.
- Track two attribution views in one sheet/dashboard: `First touch` (demand creation) and `Last touch` (demand capture).
- Tie outcomes to business funnel KPIs: `Contacts created -> Deals created -> Revenue`.
- Add one required lead-form field: “How did you hear about us?” as a qualitative check.

**Why this can work now**
- Fastest setup, minimal engineering, enough signal to cut losing channels quickly.
- Matches short-cycle service-sales reality where speed beats modeling sophistication.

**Evidence (source + evidence date)**
- HubSpot campaign attribution reports document supports attribution by contacts/deals/revenue and model selection. Evidence date: **January 2, 2026** (last updated).  
  https://knowledge.hubspot.com/campaigns/use-campaign-attribution-reports
- HubSpot attribution report types define top/mid/bottom funnel measurement (`contact create`, `deal create`, `revenue`). Evidence date: **November 24, 2025** (last updated).  
  https://knowledge.hubspot.com/reports/create-attribution-reports
- Google Analytics Help documents manual tagging dimensions from UTM parameters (`utm_source`, `utm_medium`, `utm_campaign`, etc.). Evidence date: **crawled ~December 2025**.  
  https://support.google.com/analytics/answer/11242870

### Option B — Experiment-led channel optimization stack
**What it is**
- Keep attribution simple (last-touch operationally) but run disciplined experiments every week.
- For paid channels, use controlled campaign experiments (control vs treatment arms).
- Before each experiment, define MDE, power, and stop rules to avoid false wins.

**Why this can work now**
- Faster learning loop than building advanced multi-touch first.
- Strong fit when testing hooks/offers/creative for early client acquisition.

**Evidence (source + evidence date)**
- Google Ads API experiment workflow defines control/treatment arm process for campaign testing. Evidence date: **February 26, 2026** (last updated).  
  https://developers.google.com/google-ads/api/docs/experiments/overview
- Optimizely support docs describe sequential testing/statistical methods for continuous monitoring. Evidence date: **January 22, 2026** (updated).  
  https://support.optimizely.com/hc/en-us/articles/39714777161229-Statistical-analysis-methods-overview
- Statsig documentation defines power analysis and MDE usage to plan experiment duration/traffic. Evidence date: **crawled ~December 2025**.  
  https://docs.statsig.com/experiments/power-analysis

### Option C — Warehouse-grade multi-touch attribution
**What it is**
- Centralize touch and conversion events in a warehouse.
- Run reusable attribution models (first/last/position/custom) through dbt.
- Use property-level attribution settings as governance guardrails; run a model-comparison layer for planning decisions.

**Why this can work**
- Highest analytical ceiling and better long-term channel budget allocation.
- Better for larger spend and multi-channel complexity.

**Evidence (source + evidence date)**
- Snowplow attribution dbt package tutorial (required data model inputs and setup flow). Evidence date: **January 8, 2026** (last updated).  
  https://docs.snowplow.io/tutorials/attribution/intro/
- Google Analytics Admin API attribution settings define lookback windows and reporting attribution model affecting conversion/revenue reporting. Evidence date: **April 2, 2025** (last updated).  
  https://developers.google.com/analytics/devguides/config/admin/v1/rest/v1alpha/AttributionSettings

## 3) Primary lane design (attribution model, KPI definitions, experiment loop)

### Attribution model (operational)
- Default reporting: **Dual view**
  - `First touch` for top-of-funnel channel sourcing decisions.
  - `Last touch` for near-term conversion decisions.
- Reconciliation rule:
  - If first-touch and last-touch disagree, prioritize last-touch for immediate tactical changes, first-touch for weekly budget reallocation.
- Add qualitative source check:
  - Required form field: “How did you hear about us?”
  - If mismatch rate between UTMs and self-reported source >20%, audit tracking implementation.

### KPI definitions (minimum viable set)
- `Lead Volume` = count of qualified inbound leads/day.
- `Cost per Lead (CPL)` = channel spend / qualified leads.
- `Lead-to-Call Rate` = booked intro calls / qualified leads.
- `Call-to-Close Rate` = closed-won clients / completed sales calls.
- `CAC` = total acquisition spend / new paying clients.
- `Payback (days)` = CAC / average daily gross profit per client.
- `7-day Content->Lead Conversion` = qualified leads from content clicks within 7 days / tracked content clicks.
- `Pipeline Velocity` = sum(expected deal value * win rate)/average days-to-close.

UNPROVEN: A strict 7-day window is always optimal for service funnels. Validate against your observed sales-cycle distribution.

### Experiment loop (weekly cadence)
- Unit of experimentation: one hypothesis per lane (`offer`, `creative angle`, `distribution channel`, or `CTA`).
- Pre-register for each test:
  - primary metric,
  - minimum sample threshold,
  - stop condition (timebox + minimum evidence threshold),
  - rollout rule.
- Decision policy:
  - `Win`: deploy and retest against new challenger.
  - `No decision`: extend once, then archive.
  - `Loss`: kill quickly and reallocate effort.

UNPROVEN: A fixed p-value threshold alone guarantees business impact. Pair statistical evidence with effect size and unit economics guardrails.

## 4) Key failure modes + mitigations

- UTM inconsistency causes broken channel reporting.
  - Mitigation: enforce naming schema (`source/medium/campaign/content`), lint links before publish, maintain approved value list.
- Attribution drift from missing/offline touches.
  - Mitigation: add mandatory self-reported source field and weekly mismatch audit.
- Over-optimizing for leads, not paying clients.
  - Mitigation: monitor full funnel (`lead -> call -> close -> revenue`) and gate decisions on CAC/payback.
- Premature complex modeling slows execution.
  - Mitigation: hold warehouse multi-touch until either (a) 10 clients closed or (b) channel spend justifies added complexity.
- False positives from underpowered experiments.
  - Mitigation: run power/MDE checks before launch and use fixed stop rules.

## 5) Concrete 14-day execution outline

### Days 1-2: instrumentation baseline
- Define UTM taxonomy and publish one-page tracking SOP.
- Add required lead field: “How did you hear about us?”
- Create funnel stages in CRM: `Lead`, `Qualified`, `Call Booked`, `Proposal`, `Closed Won`.

### Days 3-4: dashboard + QA
- Build daily dashboard (or sheet) with first-touch + last-touch views.
- Add KPI formulas (CPL, CAC, lead-to-call, call-to-close, payback).
- QA 10 end-to-end test clicks across Reddit/Instagram/outbound links.

### Days 5-7: baseline data collection
- Run current channels with no major creative changes (establish baseline).
- Tag every content/post/outbound link with approved UTMs.
- Daily check: missing UTM rate, unattributed lead rate, self-reported mismatch rate.

### Days 8-10: experiment sprint 1
- Launch 2 tests only:
  - Test A: offer framing (e.g., “done-for-you posting” vs “pipeline-focused content ops”).
  - Test B: CTA format (audit call vs sample-content teardown).
- Pre-register metrics and stop criteria before launch.

### Days 11-12: evaluate + reallocate
- Evaluate using both attribution views and funnel progression.
- Keep/kill decisions based on CAC trend + downstream conversion impact.
- Shift effort to top 1-2 channels by `qualified leads` and `call-to-close`.

### Days 13-14: experiment sprint 2 setup
- Ship winning variants from sprint 1 as new control.
- Queue next 2 hypotheses (new hook + new audience segment).
- Produce weekly measurement memo: what changed, why, and expected impact next week.

UNPROVEN: Hitting first 10 paying clients within 14 days is generally achievable; timeline depends heavily on existing audience trust, offer-market fit, and sales-cycle length.
