# L06 Measurement System

Primary focus: attribution model, KPI definitions, and experiment loop for fastest path to first 10 paying clients.

## 1) Three in-lane options

### Option A: Lean Hybrid Attribution (recommended for first 10 clients)

What it is:
- Use HubSpot `Original Traffic Source` as first-touch and `Latest Traffic Source` as near-last-touch.
- Enforce strict UTM naming on every outbound/link-in-bio/reddit/IG link.
- Add one required intake field on booking form: "How did you hear about us?" (self-reported attribution).
- Weekly manual reconciliation: self-reported vs tracked source, then decide next week’s tests.

Attribution model:
- Decision model = `60% self-reported source + 40% latest tracked source` for channel budgeting.
- UNPROVEN: 60/40 weighting is a practical starting heuristic for dark-social-heavy service sales, but this ratio is not validated by a controlled trial in this business yet.

Core KPI definitions:
- `Lead Volume` = count of qualified inbound leads/week.
- `Lead->Call Rate` = booked calls / qualified leads.
- `Call->Client Close Rate` = paying clients / sales calls.
- `CAC (channel)` = channel spend / paying clients from channel.
- `Time-to-Close` = median days from lead created to payment.
- `Payback (proxy)` = days to recover CAC from first invoice.

Experiment loop:
- Weekly cycle: hypothesis -> launch 1-2 channel/offer tests -> Friday readout -> keep/kill/iterate.
- Guardrails: no experiment scales if `Call->Client Close Rate` drops >20% vs previous week.

Evidence sources (with evidence date):
- HubSpot traffic source properties (defines first/latest source + UTM drill-down behavior). Evidence date: **Last updated February 2, 2026**. https://knowledge.hubspot.com/properties/understand-traffic-source-properties
- HubSpot campaign attribution reports (contacts/deals/revenue attribution concepts). Evidence date: **Last updated January 2, 2026**. https://knowledge.hubspot.com/campaigns/use-campaign-attribution-reports
- Microsoft "Dirty Dozen" (metric interpretation pitfalls; supports guardrails/interpretation discipline). Evidence date: **KDD August 2017**. https://www.microsoft.com/en-us/research/publication/a-dirty-dozen-twelve-common-metric-interpretation-pitfalls-in-online-controlled-experiments/

### Option B: HubSpot Multi-Touch Reporting (professionalized but slower)

What it is:
- Use HubSpot attribution report builder with multiple attribution models in parallel (e.g., first-touch, last-touch, and weighted model views).
- Maintain full funnel KPI stack: contact create, deal create, and revenue attribution.
- Formal weekly experiment review with model-by-model deltas.

Attribution model:
- Compare model outputs side-by-side weekly; allocate next-week effort by consensus across models.
- UNPROVEN: averaging model outputs improves decision quality for this specific service niche; validate after 4-6 weeks.

Core KPI definitions:
- `Top-funnel Attribution` = contact create contribution by channel.
- `Mid-funnel Attribution` = deal create contribution by channel.
- `Bottom-funnel Attribution` = won revenue by channel.
- `Pipeline Velocity` = won deals / average days-in-pipeline.
- `Channel Efficiency Index` = attributed revenue / channel spend.

Experiment loop:
- 7-day rhythm with mandatory pre-registered hypothesis and success metric.
- Decisions: ship, iterate, or stop based on primary metric + no guardrail violations.

Evidence sources (with evidence date):
- HubSpot attribution report builder (contact/deal/revenue attribution report types). Evidence date: **Last updated November 24, 2025**. https://knowledge.hubspot.com/reports/create-attribution-reports
- HubSpot campaign attribution reports (attribution model selection and discrepancy notes). Evidence date: **Last updated January 2, 2026**. https://knowledge.hubspot.com/campaigns/use-campaign-attribution-reports
- Microsoft ExP Platform paper (trustworthy experimentation systems and rapid readout principles). Evidence date: **ICSA April 2018**. https://www.microsoft.com/en-us/research/publication/the-anatomy-of-a-large-scale-experimentation-platform/

### Option C: Experimentation-First Stack (highest rigor, lowest short-term speed)

What it is:
- Stand up a dedicated experimentation decision framework (primary + guardrail metrics), then join outcomes to CRM revenue.
- Run near-continuous tests across offers, hooks, landing pages, and outbound scripts.

Attribution model:
- Exposure-based experimental attribution for conversion lift, with CRM revenue as downstream validation.
- UNPROVEN: this overhead pays off before client #10 in a small service business; likely better after initial PMF traction.

Core KPI definitions:
- `Primary` = incremental booked calls per visitor (or per outbound message).
- `Guardrails` = lead quality score, show-up rate, refund/churn proxy.
- `Decision SLA` = days from launch to statistically interpretable call.

Experiment loop:
- Template-driven setup with explicit ship/no-ship rules and reviewer override path.

## 2) Lane winner with explicit score math (0-100)

Scoring criteria (weighted to "first 10 paying clients"):
- Speed to usable insights: 40
- Setup effort/time: 20
- Attribution fidelity: 20
- Experiment rigor: 20

Scoring method:
- Each criterion scored 0-10.
- Weighted score contribution = `(criterion score / 10) * criterion weight`.
- Total score = sum of weighted contributions (0-100).

Option A score:
- Speed 9/10 -> 36
- Setup 9/10 -> 18
- Fidelity 5/10 -> 10
- Rigor 6/10 -> 12
- **Total = 36 + 18 + 10 + 12 = 76/100**

Option B score:
- Speed 7/10 -> 28
- Setup 6/10 -> 12
- Fidelity 8/10 -> 16
- Rigor 7/10 -> 14
- **Total = 28 + 12 + 16 + 14 = 70/100**

Option C score:
- Speed 3/10 -> 12
- Setup 3/10 -> 6
- Fidelity 9/10 -> 18
- Rigor 9/10 -> 18
- **Total = 12 + 6 + 18 + 18 = 54/100**

Winner:
- **Option A (76/100)** for immediate execution and fastest learning toward first 10 clients.

## 3) Key failure modes + mitigations

1. Attribution gaps from dark social / DMs / copied links.
- Mitigation: mandatory self-reported source field; weekly reconciliation; preserve both tracked and self-reported source in CRM.

2. UTM inconsistency breaks channel reporting.
- Mitigation: single UTM taxonomy doc + link builder; reject non-compliant campaign links.

3. Vanity metrics (views/likes) displace pipeline metrics.
- Mitigation: decision dashboard starts with paying-client funnel KPIs only; reach metrics are secondary.

4. Low sample size causes false conclusions.
- Mitigation: run fewer, bigger tests; require minimum sample thresholds before channel budget shifts.

5. Team skips readouts and repeats failed tests.
- Mitigation: fixed Friday readout with experiment registry (hypothesis, owner, decision, next action).

## 4) Concrete 14-day execution outline

Day 1:
- Lock KPI dictionary (exact formulas above) in one shared doc.
- Set weekly target for first 10-client sprint (leads, calls, closes).

Day 2:
- Define UTM convention: `utm_source`, `utm_medium`, `utm_campaign`, `utm_content`.
- Create approved channel/source value list (IG, Reddit, outbound DM, referral, etc.).

Day 3:
- Add required "How did you hear about us?" field on all lead capture forms.
- Add a required sales-call note template with source confirmation.

Day 4:
- Build CRM properties and views: first source, latest source, self-reported source, qualified lead flag, close outcome.
- Create one funnel dashboard: Leads -> Calls -> Clients.

Day 5:
- Instrument all active links (bio links, Reddit links, outbound profile links) with UTMs.
- QA 10 end-to-end clicks and verify source fields populate.

Day 6:
- Define experiment backlog (at least 6 tests) across offer, hook, channel, and CTA.
- Prioritize top 2 tests by expected impact and ease.

Day 7:
- Launch test #1 (offer framing) and test #2 (channel message angle).
- Record hypothesis, primary KPI, guardrail KPI, start date.

Day 8:
- Midweek data hygiene check: missing UTMs, missing source fields, duplicate leads.
- Fix instrumentation issues immediately.

Day 9:
- Continue traffic/outbound volume; do not change test variables mid-run.
- Start drafting next test pair based on early directional signals.

Day 10:
- Pull interim cohort snapshot (by source + self-report).
- Flag anomalies; mark any causal interpretation as **UNPROVEN** if under threshold.

Day 11:
- Prep weekly readout template: outcome, confidence, decision, next action.
- Precompute CAC and close-rate deltas by channel.

Day 12:
- Run Friday readout; decide keep/kill/iterate for each test.
- Reallocate next-week effort to top 1-2 channels.

Day 13:
- Launch next iteration using winning angle(s).
- Archive losing variants and reason codes.

Day 14:
- Executive checkpoint: score progress toward first 10 clients, update forecast, confirm next 14-day test plan.
