# L05: Conversion Stack

## 1) Three in-lane options

### Option A (Speed-first): Carrd + Tally + HubSpot Free CRM + Make

**Best use case:** Fastest path to first 10 paying clients with low fixed cost and minimal setup overhead.

**Landing stack**
- 1-page Carrd offer page (headline, proof, package tiers, CTA).
- Tally embedded qualification form + hidden UTM fields.
- Thank-you page with instant calendar CTA or DM instructions.

**CRM schema (HubSpot properties to create now)**
- `lead_source` (enum: Reddit, Instagram, Referral, Other)
- `utm_source`, `utm_medium`, `utm_campaign` (text)
- `offer_variant` (enum)
- `service_interest` (enum: creation-only, creation+posting, monthly retainer)
- `monthly_budget_band` (enum)
- `timeline` (enum)
- `qualified` (boolean)
- `pipeline_stage` (enum: New, Qualified, Booked, Proposal, Won, Lost)
- `next_action_at` (datetime)
- `owner` (user)

**Automation handoff points**
- Tally submission -> Make scenario -> create/update HubSpot contact + deal.
- If `qualified=true`, auto-create task and send booking link.
- If no booking in 24h, trigger follow-up email/DM task.
- On `Won`, webhook into your ops board to start onboarding + content intake.

## 3) Key failure modes + mitigations

- Failure mode: Lead capture works but follow-up is inconsistent.
- Mitigation: Enforce 15-minute SLA task on every inbound lead; daily missed-SLA report.

- Failure mode: Low-intent leads flood pipeline.
- Mitigation: Add hard qualification fields (budget, timeline, channel volume) and auto-mark `qualified=false` when thresholds fail.

- Failure mode: Attribution is unclear across Reddit/Instagram.
- Mitigation: Require hidden UTMs and preserve first-touch + last-touch fields in CRM.

- Failure mode: Handoffs to onboarding are manual and delayed.
- Mitigation: Use a single `Won` event webhook that creates onboarding task bundle automatically.

- Failure mode: Over-automating before message/offer fit is proven.
- Mitigation: Keep only 3 automations in week 1 (capture, follow-up, won-handoff); expand after 10 closed-loop sales conversations.


## 4) Concrete 14-day execution outline

### Days 1-2
- Finalize one core offer + one upsell + one guarantee.
- Build Carrd page with one CTA and one qualification form path.
- Create HubSpot properties and pipeline stages exactly as listed above.

### Days 3-4
- Build Tally form (with hidden UTMs).
- Connect Tally -> Make -> HubSpot create/update logic.
- Test 10 synthetic submissions and verify dedupe + stage assignment.

### Days 5-6
- Build follow-up automation: immediate confirmation + 24h reminder + owner task.
- Build `Won` handoff webhook to onboarding board/checklist.
- QA all automations with test contacts.

### Days 7-8
- Publish landing page.
- Launch outbound/inbound traffic from Reddit + Instagram CTAs.
- Start daily dashboard: leads, qualified %, booked %, proposal %, close %.

### Days 9-10
- Review first conversion bottleneck (form completion vs booking vs close).
- Tighten form fields and rewrite CTA/offer block.
- Claim that multi-step form will increase conversion is **UNPROVEN** until A/B data exists in this funnel.

### Days 11-12
- Add one retargeting/remarketing follow-up path for non-booked qualified leads.
- Add one referral ask automation for positive-call outcomes.

### Days 13-14
- Run funnel retrospective on first 2 weeks of leads.
- Freeze winning message + qualification thresholds.
- Produce v2 pipeline rules and weekly operating cadence for continued scale to clients 11-25.
- Claim that this exact sequence guarantees 10 clients in 14 days is **UNPROVEN** (depends on traffic quality, offer-market fit, and close rate).
