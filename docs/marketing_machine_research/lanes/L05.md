# L05 - Conversion Stack (Landing Stack + CRM Schema + Automation Handoffs)

## 1) Three in-lane options

### Option A - Lean no-code speed stack (recommended for first 10 clients)
**Landing stack**
- One-page service LP (hero offer, proof, CTA) + embedded qualification form.
- Primary CTA: "Book strategy call" via Calendly; backup CTA: "Start now" via Stripe Payment Link.

**CRM schema (Airtable base)**
- `Leads`: `lead_id`, `created_at`, `source_channel` (Reddit/IG/referral), `offer_interest`, `budget_band`, `urgency`, `status`.
- `Contacts`: `contact_id`, `lead_id`, `name`, `email`, `IG_handle`, `timezone`.
- `Deals`: `deal_id`, `lead_id`, `package_type`, `price_quote`, `stage`, `close_date`, `payment_link`.
- `Activities`: `activity_id`, `lead_id`, `type` (form/call/DM/payment), `timestamp`, `owner`, `notes`.

**Automation handoff points**
- Form submit -> Zapier -> create/update Airtable `Leads` + `Contacts`.
- Qualified lead -> Calendly booking link (or Calendly Routing Form for team routing).
- Meeting booked/no-show/canceled -> Zapier -> update `Activities` + `Deals.stage`.
- Closed-won -> Stripe payment success webhook -> update `Deals.stage=Paid` + trigger onboarding email.

**Why it can win fast**
- Low setup overhead and no-code payment collection.
- Supports immediate speed-to-lead workflows (critical for inbound close rate).

### Option B - HubSpot-centered stack (single system of record)
**Landing stack**
- HubSpot form + HubSpot meetings scheduler page on offer landing page.
- Optional direct checkout link post-call (Stripe Payment Link).

**CRM schema (HubSpot objects)**
- `Contact`: lifecycle stage, source, service interest, monthly content volume, timeline.
- `Company` (if B2B): industry, team size, social footprint.
- `Deal`: pipeline (`New lead -> Qualified -> Call booked -> Proposal -> Closed won/lost`), amount, close date.
- `Activities`: calls, meeting outcomes, follow-up tasks, email events.

**Automation handoff points**
- Form submit -> contact create/update -> auto-enroll workflow.
- Workflow branches by budget/urgency -> book meeting CTA + owner assignment.
- Meeting events -> deal stage transition + follow-up sequence.
- Payment confirmation (via integration/Zap) -> `Closed Won` + onboarding task creation.

**Why it can win**
- Fewer integration seams; robust native reporting.
- Better for scaling after first 10 clients.

**Tradeoff**
- Higher seat and feature gating can slow early execution.

### Option C - Conversational qualification stack (Typeform-first intake)
**Landing stack**
- Typeform multi-step qualifier as front door.
- High-fit respondents routed to Calendly; low-fit respondents get nurture flow.

**CRM schema (HubSpot or Airtable-backed)**
- `Lead Fit`: score, intent reason, content goals, posting cadence need.
- `Acquisition Meta`: UTM fields, channel, ad/content asset ID.
- `Pipeline`: MQL, SQL, Discovery Booked, Proposal, Won/Lost.

**Automation handoff points**
- Typeform submit (including partial submit if enabled) -> HubSpot contact/deal update.
- Score threshold met -> instant Calendly scheduling link.
- No booking within 15 minutes -> auto reminder + manual reach-out task.

**Why it can win**
- Better lead qualification before calls, can reduce low-fit meeting load.

**Tradeoff**
- More moving parts and potential drop-off in multi-step forms; impact depends on form quality and traffic source (**UNPROVEN** for your funnel until tested).

## 3) Key failure modes + mitigations

- Failure mode: Slow lead follow-up (>5-15 min) kills qualification odds.
  - Mitigation: Trigger instant auto-reply + owner alert + SLA timer; if no human response in 10 minutes, escalate to fallback owner.

- Failure mode: No-shows on booked calls.
  - Mitigation: Enable Calendly reminder/reconfirmation workflows + 1-click reschedule link + same-day SMS reminder.

- Failure mode: Data fragmentation (LP/form, scheduler, CRM, payments not reconciled).
  - Mitigation: Use one canonical `lead_id`; write all events to `Activities`; daily reconciliation query for orphaned payments/bookings.

- Failure mode: Form friction lowers conversion.
  - Mitigation: Keep first-step form fields <= 4 and defer deeper qualification to post-submit or call; exact uplift by field count is **UNPROVEN** for your audience until A/B tested.

- Failure mode: Over-automation produces generic outreach.
  - Mitigation: Use templated personalization tokens (channel + pain point + asset referenced); enforce manual review on high-ticket leads.

## 4) Concrete 14-day execution outline

### Days 1-3 (build MVP stack)
- Day 1: Define one core offer, one guarantee, one CTA path (Book Call) and one fallback CTA (Pay Deposit).
- Day 2: Build LP + form + thank-you page + Stripe Payment Link.
- Day 3: Create Airtable schema (`Leads/Contacts/Deals/Activities`) and basic dashboard (new leads, booked, show rate, close rate).

### Days 4-6 (wire automations)
- Day 4: Implement form -> Airtable + auto-email + owner alert.
- Day 5: Implement Calendly booked/rescheduled/no-show -> Airtable stage transitions.
- Day 6: Implement payment success -> `Closed Won` + onboarding kickoff automation.

### Days 7-9 (launch traffic + outbound)
- Day 7: Publish LP and QA on mobile/desktop; test all handoffs with 5 end-to-end test leads.
- Day 8: Start daily outbound/inbound capture from Reddit + Instagram with UTM-tagged links.
- Day 9: Activate follow-up cadence: T+10 min, T+24h, T+72h for non-booked leads.

### Days 10-12 (optimize conversion)
- Day 10: Review funnel metrics by source: visitor -> form -> booked -> showed -> paid.
- Day 11: Rewrite LP headline/offer and shorten form if booked-rate < target.
- Day 12: Add objection-handling assets (mini case proof, pricing FAQ, process timeline).

### Days 13-14 (close sprint)
- Day 13: Run a concentrated close sprint on all open SQLs (same-day follow-ups + deadline offer).
- Day 14: Retro and decide keep/kill rules for channels, then lock the next 14-day test plan.

**Minimum operating targets for first sprint (practical, not guaranteed):**
- Lead response median <10 minutes.
- Show rate >=60%.
- Close rate from attended calls >=20%.

Target achievability at these exact thresholds is **UNPROVEN** until you run one full 14-day cycle with your real offer, traffic mix, and pricing.
